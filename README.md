# 🚀 Inventory & Demand Forecasting System

This is a project I built to solve a real problem — small manufacturers often struggle to manage inventory and don’t have any way to predict future demand.

So I created a simple full-stack application that helps track products, manage stock, and even gives basic demand predictions using data.

---

## 💡 What this project does

* Lets users create an account and log in
* Add and manage products (stock, updates, delete)
* Track sales data
* Show a dashboard with useful insights
* Predict future demand (next few days) using simple ML

---

## 🛠️ Tech I used

**Backend:** FastAPI, SQLAlchemy
**Frontend:** React
**Database:** PostgreSQL (Supabase)
**ML:** Python (Pandas, scikit-learn)

---

## 🧠 Why I built this

I wanted to move beyond just tutorials and build something practical.

Most small businesses still manage inventory manually or on Excel. This project is a small step towards solving that using tech + data.

Also, I wanted to:

* Understand how frontend + backend actually connect
* Work with real APIs and databases
* Try applying machine learning to a real use case

---

## ⚙️ How to run locally

Clone the repo:

git clone https://github.com/your-username/inventory-ai-saas.git

### Backend

cd backend
pip install -r requirements.txt
uvicorn main:app --reload

### Frontend

cd frontend
npm install
npm start

## 🌐 Deployment

* Backend hosted on Render
* Frontend hosted on Netlify/Vercel
